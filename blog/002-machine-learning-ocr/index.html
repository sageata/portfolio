Automating the Onboard Cabin Report: A Machine Learning Approach

Onboard Cabin Reports Classification using ML

When I joined the Guest Delivery team, now known as Inflight Service & Hospitality, I quickly noticed an over-reliance on Excel. Don’t get me wrong—I love Excel—but it’s often misused.

The Challenge

The Onboard Cabin Report (OCR) is a crucial document authored by Senior Crew Members. It serves as a communication channel for various business-related issues, such as guest complaints, defects, and shortages. Previously, these reports were manually classified by a few individuals who were already swamped with other tasks. The process involved receiving a CSV file of the previous day’s reports, copying the entries into an Excel sheet, and then manually categorizing each report using a three-level hierarchical taxonomy. This was not only time-consuming but also error-prone.

The Turning Point

I felt that our time could be better utilized. So, I decided to learn how to train a machine learning model for this task. I gathered historical reports, cleaned the data, and prepared it for training. I used spaCy, a Natural Language Processing library for Python, to fine-tune the model, achieving a 92% accuracy rate in classification.

The Solution

To make the solution truly effective, I had to develop additional components. I automated data consolidation using Power Automate and created a custom Azure function to run the Python code for classification. This function added the classification columns and data validation back into an Excel file, making the entire process seamless.

Key Outcomes

  • Error Reduction: Significantly reduced errors during the classification process.
  • Data Completeness: Ensured the OCR data was complete, as the classification was done automatically in the background.
  • Automated Reporting: Created a Power BI report that used the Excel file as a source and refreshed automatically.
