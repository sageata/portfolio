Onboard Cabin Reports Classification using ML

When I joined the Guest Delivery team (now Inflight Service & Hospitality) I’ve encountered the over reliance on Excel. Don’t get me wrong, I love Excel, but people use it wrong all the time.

The Onboard Cabin Report (OCR for short) is a report authored by the Senior Crew Members that convey different messages to the business, such as guest complaints, defects, shortages, you get the idea.

These reports used be classified manually by a handful of people that had 1000 other things to do. The process was something like this, one person was receiving a CSV file with the reports submitted the previous day, this person copy-pasted the entries into an Excel file and then added additional columns that classified the report with a 3 levels deep hierarchical taxonomy. Each report was read and people would refer to a different sheet and copy-pasted the category of the report. At the end of the month a report bas built manually based on this classification, from scratch.

Once again, I felt that our time could be better spent on something else, therefore I’ve decided to embark on learning how to train a machine learning model. I’ve gathered all the reports from the previous years that have been classified this way, cleaned it and prepared the dataset for training. Used spaCy, which is a Natural language processing library for Python that offers tokenization, and parsing, and after fine-tuning the model achieved a 92% accuracy on the classification.

To make the solution compelling I had to develop a few other things along the way, such a way of automating the data consolidation using Power Automate, and since Power Automate cannot run Python code I had to create a custom Azure function that was doing the inference of the categories based on the text written in the OCRs and added the classification columns along with data validation back into an Excel file.

Key Outcomes

  • reduced the number of errors caused during the classification
  • assured completeness of the OCR data since this was done automatically in the background
  • been able to generate a Power BI report using the Excel file as a source which refreshed automatically
  • powerautomate, spaCy, azurefunctions, powerbi
